Developer Resources

Open source libraries, discussions, news and more about the Open Permit APIs and PRDF.

Overview

This page contains a set of developer resources to help you get started with Open Permit development.

Mailing List

The mailing list is where most communication will happen. Please join the mailing list if you are doing any work around Open Permit or if you have any questions. Please contact us at support@epermithub.org if you have problems.

Twitter

Relevant developer news will also be broadcast via the Open Permit Twitter account.

Code Repositories

The API specs and PRDF documentation as well as code libraries and other source code will be distributed via the Open Permit account on Github. Reference to other third party code and other resources will be referenced from this page.

Issue Tracking

The Open Permit issue tracker at https://github.com/openpermit/openpermit.github.io/issues is primarily used to track the development of the Open Permit APIs and PRDF specs. After issues or a new feature are discussed on the mailing list, tickets should be open here to ensure that they get addressed in a future milestone. This is not the place to report issues with the code libraries or specific implementations of Open Permit. If you find a problem with a specific implementation, please help notify the owner of the endpoint or let us know at support@epermithub.org and we will make sure to bring it up to the right organization.

openpermit.js

Introduction

openpermit.js is a Javascript and Node.js library that provides wrappers around the Open Permit API and PRDF JSON support. The library allows you to filter permit requirement checklists, calculate fees and wraps API calls to make them easy to invoke. It also provides utility functions for common operations such as retrieving values from UI, etc.

Releases

Currently openpermit.js is at v0.3.0. The library is under heavy research and development and the API is very unstable. We encourage you to download and play with it, but do not expect the API to remain consistent.

Even better if you would like to contribute or have any feedback, head over to the GitHub repo or get in contact with us at support@epermithub.org.
